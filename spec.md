# 監理・整理銘柄一覧取得仕様

## 概要

`scrape_jpx_supervision_history.py` は、JPX の監理・整理銘柄指定履歴を取得し、`supervision.csv` を生成します。  
差分更新用に `state.json` を管理します。

取得対象ページ:

- `https://www.jpx.co.jp/listing/market-alerts/supervision/01.html`
- 現行ページ内から見つかった `01-archives-XX.html`

## 入出力ファイル

- 入力元
  - JPX の指定履歴ページ
- 出力
  - `supervision.csv`
  - `state.json`

`supervision.csv` と `state.json` は実行時に生成するファイルで、リポジトリには含めません。

## 出力CSV仕様

ファイル名:

- `supervision.csv`

文字コード:

- `cp932`

列順:

1. `コード`
2. `市場`
3. `日付`
4. `日付`
5. `内容` ごとの理由列

列内容:

- `コード`
  - JPX の銘柄コード
  - 数字4桁または英字付き4桁を文字列で出力
- `市場`
  - 固定値 `t`
- 3列目 `日付`
  - 指定年月日
- 4列目 `日付`
  - 解除年月日の前営業日
  - 解除年月日がない場合は `2099/12/31`
- 5列目以降
  - JPX の `内容` 原文を列名として使用
  - 該当理由の列に `1`
  - 非該当理由の列に `0`

初回生成時の理由列:

- 出力CSVが存在しない場合は対象URLを全件巡回して `内容` の種類を収集する
- 収集した理由を5列目以降の列として出力する

差分更新時の理由列:

- 既存CSVの理由列は維持する
- 新しい `内容` が見つかった場合は末尾に列を追加する

日付形式:

- `YYYY/MM/DD`

ソート:

- `コード` 昇順
- 指定年月日昇順
- 解除前営業日昇順

重複排除:

- `コード`
- 指定年月日
- 解除前営業日
- 理由列は同一キー内で OR マージする

継続中理由の引き継ぎ:

- 同一コードで後続の指定年月日が現れた時点で、前行の終了前営業日がその指定年月日以降なら前行は継続中とみなす
- 継続中の前行は、後続行の指定年月日の前営業日で終了するように終了前営業日を補正する
- 後続行の理由列は、後続行自身の理由と、その指定日時点で継続中だった前行の理由を OR して出力する

## state.json 仕様

保存形式:

```json
{
  "last_date": "2026-04-17",
  "last_code": "4530"
}
```

意味:

- `last_date`
  - 保存済みCSV内で最大の指定年月日
- `last_code`
  - `last_date` 内で最大のコード

## 差分更新仕様

通常実行時の取得条件:

- `指定日 > last_date` の行を取得
- `指定日 == last_date` の場合は `コード > last_code` の行を取得
- `指定日 == last_date` かつ `コード <= last_code` はスキップ

state 更新基準:

- 保存済み `supervision.csv` を基準に更新
- 保存成功後に 1 回だけ更新
- 出力CSVが存在しない場合は `state.json` を無視して初回生成扱いにする

壊れた state:

- warning を出す
- 初回扱いで続行する

## 取得・抽出ルール

- `01.html` を起点に履歴ページURLを動的検出する
- `option value` から `01.html` と `01-archives-XX.html` を優先抽出する
- `option value` で取れない場合はリンクやHTML文字列から同URLを抽出する
- 履歴ページが見つからない場合のみ `01-archives-01.html` から連番探索する
- `指定履歴` セクションの表を対象にする
- 表が直接見つからない場合は、必要列を含む最有力表にフォールバックする
- 列名揺れは正規化して吸収する
- 行の列数が不足している場合は空欄で補完する
- 行の列数が多い場合は想定列数まで切り詰める
- 明らかに壊れた行は warning を出してスキップする

対象判定:

- `内容` に `監理` または `整理` を含む行
- `市場区分` に `外国株` を含む行は除外

継続条件:

- HTTP エラー時はその URL のみ warning を出して継続
- 1 ページ失敗しても他ページは継続

## 前営業日ルール

- 解除年月日の 1 営業日前を出力する
- 土日を除外する
- JPX 休業日を除外する
- 解除年月日が空欄または `-` の場合は `2099/12/31`

## 実行時表示

標準出力:

- 総件数
- 保存先
- 年ごとの取得件数

warning 出力:

- 取得失敗 URL
- 壊れた行
- 壊れた `state.json`

## 注意点

- `--full` は既存 `supervision.csv` を再利用せず、取得結果で再生成する
- 4列目が `2099/12/31` の行は、JPX 側で解除年月日が未設定の行
- 年ごとの取得件数は取得できたページの `指定年月日` から集計する
